# MIDI Chord Lab — プロジェクト概要

**バージョン:** 2.10.0  
**ライセンス:** MIT  
**種別:** Windows 向けデスクトップアプリ（開発時は Python、配布は PyInstaller exe）

MIDI ファイルから和声（コード）を抽出・編集し、理論アシスト・可視化分析・再生・書き出しまでを一つのワークステーションにまとめたアプリケーションです。作曲・編曲・和声学習の補助ツールとして、**ローカル完結**（クラウド API 不要）で動作します。

---

## 解決する課題

| 課題 | 本アプリのアプローチ |
|------|----------------------|
| MIDI からコード進行を把握しづらい | 和声タイムラインでコード・音名・ローマ数字を一覧・編集 |
| DAW だけでは理論的な置換候補が出ない | キーに基づく**ルールベース理論アシスト** |
| 声部の動きを追いにくい | **声部進行解析**（連続和音間の移動量・跳躍の可視化） |
| 演奏傾向の把握に時間がかかる | **パフォーマンス統計**とダッシュボード |
| 成果物の共有形式がバラバラ | MusicXML / MIDI / HTML 分析レポートの書き出し |

## 技術スタック

| 領域 | 採用技術 |
|------|----------|
| UI | PyQt6（3 ペイン、テーマ・デザイントークン） |
| 音楽理論・解析 | music21 |
| 可視化 | matplotlib（ピアノロール、パフォーマンス、声部進行） |
| MIDI I/O | mido |
| 音声出力 | sounddevice（ソフトウェア音源） / Windows MIDI デバイス |
| 数値処理 | NumPy（声部割当・波形ミックス等） |
| 配布 | PyInstaller、Inno Setup（インストーラー） |

**必要環境:** Python 3.11 以降（開発）、Windows 10/11（配布 exe）

### MIDI ビジュアライザ（`rust/midi_viz`）— GPU 中心設計

| 層 | 内容 |
|----|------|
| インスタンス | CPU は可視ノートのメタのみ転送（相対拍 `onset_rel` / `end_rel`、MIDI、ベロシティ、チャンネル、`kind`）。矩形 `rect_min/max` は廃止。 |
| 頂点シェーダー | `note.wgsl` がウォーターフォール落下、白鍵／黒鍵の X 配置、円形レイアウト、サイバー／スペクトラム用オーバーレイをスタイル別に算出。 |
| フラグメント | HSV→RGB、ベロシティに応じた発光（HDR 風加算）。 |
| シーン uniform | `NoteSceneUniform`（拍窓、`t_rel`、鍵域、スタイル等）＋スペクトラム用ストレージ（128 鍵 × height/peak/vel）。 |
| パーティクル | `particle_compute.wgsl` で全スロット物理更新。`particle_emit.wgsl` が 128 鍵のベロシティ立ち上がりを検出し GPU でスポーン（`atomicAdd` でスロット割当）。 |
| MIDI 入力 | `MidiRingBuffer`（ロックフリー）+ `live_vel[128]` で演奏ベロシティを GPU emit に合成。 |
| スタイル遷移 | `style_prev` / `style_blend` で Waterfall↔Circular 等を VS 内 `mix`（約 0.4s）。 |
| 動画出力 | `export_video_ffmpeg` — 決定論クロック + `FfmpegWriterThread` で stdin へ非同期書き込み。 |

## 主な機能

### ファイルと編集

- MIDI の読み込み（ファイル選択・ドラッグ＆ドロップ・最近使ったファイル・関連付け起動）
- **和声タイムライン** — コード／音名・長さ（拍）の編集（拍・開始位置は表示専用）
- 行の追加・複製・削除（コンテキストメニュー）
- **Undo / Redo**（`Ctrl+Z` / `Ctrl+Y`）
- 上書き保存（未保存時はタイトルに `*`）

### 分析スタジオ

- **ピアノロール** — ノートイベントの可視化
- **パフォーマンス** — ベロシティ・音域・密度などの統計とダッシュボード
- **声部進行** — 連続する和音イベント間の声部移動（非同期ワーカーで描画）
- **MIDI ビジュアライザ** — Rust/wgpu オフスクリーン描画、Waterfall / Circular / Spectrum / Cyber、Transport 同期、MIDI In、MP4・PNG 連番

### 理論・推定

- 構成音からのコード記号推定（`Cmaj7` / `Am7` など）
- **理論アシスト** — メジャー／マイナーキー向けルールベース置換候補（ダブルクリックで反映）
- 機能和声ラベル（ローマ数字表記）

### 再生・書き出し

- 再生（MIDI デバイスまたはソフトウェア音源、ファイル BPM 反映）
- 再生中のタイムライン行ハイライト
- MusicXML / MIDI / HTML 分析レポートの書き出し

### 配布・ OS 連携

- PyInstaller による単体 exe
- Inno Setup インストーラー（デスクトップショートカット、`.mid` / `.midi` 関連付け）
- コマンドライン・メニューからの MIDI ファイル関連付け登録

### `core/` の役割（注目モジュール）

| モジュール | 役割 |
|------------|------|
| `harmony.py` | 和声解析、コード記号推定（music21 + 自前テンプレート・キャッシュ） |
| `chord_rules.py` | 機能和声に基づくルールベース置換候補 |
| `voice_leading.py` | 連続和音間の声部進行（小和音は DP、大和音は貪欲法） |
| `performance_analytics.py` | 演奏統計と matplotlib ダッシュボード |
| `playback.py` | 絶対時刻スケジュールによる再生 |
| `analysis_build_worker.py` | UI をブロックしない分析の非同期実行 |
| `edit_history.py` | タイムライン編集の Undo/Redo |
| `pianoroll_plot.py` / `plotting.py` | 分析用グラフ生成 |

### `ui/` の役割

| モジュール | 役割 |
|------------|------|
| `main_window.py` | メインウィンドウ（3 ペイン、メニュー、ショートカット統合） |
| `theme.py` / `design_tokens.py` | ダークテーマ・一貫したビジュアル |
| `widgets/timeline_panel.py` | 和声タイムライン表 |
| `widgets/piano_keyboard.py` | 仮想鍵盤 |
| `widgets/voice_leading_panel.py` | 声部進行表示 |

## 注目すべき点

### 1. ルールベースの理論アシスト（独自ドメインロジック）

`chord_rules.py` にメジャーキー向けの**機能置換マトリクス**をデータとして定義し、マイナーキーは相対長調のルールで候補を生成します。LLM ではなく、音楽理論のルールをコードに落とした部分は、ポートフォリオや面接で「仕様 → 実装」を語りやすい箇所です。

### 2. コード推定の多層フォールバック

`harmony.py` では music21 の和音解析を使いつつ、失敗時や誤判定（例: 3 度があるのに power 表記）を避けるため、**インターバルテンプレート照合**と**結果キャッシュ**を実装しています。外部ライブラリ任せにせず、品質を上げるための自前レイヤーがあります。

### 3. 声部進行の割当アルゴリズム

`voice_leading.py` では、和音の音数が増えると `itertools.permutations` が爆発するため、**8 音以下はビットマスク DP**、それ以上は**貪欲法**で最小移動量の割当を行います。音楽アプリでありながら、計算量を意識した実装になっています。

### 4. 重い処理の非同期化

MIDI 読み込み（`load_worker.py`）や分析ビルド（`analysis_build_worker.py`）をワーカースレッドで実行し、UI の応答性を保ちます。ピアノロール・パフォーマンス・声部進行の再描画も、この仕組みと連動します。

### 5. プロダクト級の UI と配布

- ウェルカム画面、スプラッシュ、ローディングオーバーレイ
- デザイントークンに基づくテーマ（`theme.py` は 700 行超のスタイル定義）
- 全画面（`F11`）、豊富なショートカット
- **exe + インストーラー + ファイル関連付け**まで揃えており、「動くプロトタイプ」ではなく**配布可能な製品**として仕上げられています

### 6. ローカル完結・プライバシー

API キーやネットワーク不要で、MIDI を端末内だけで処理できます。個人の作曲データを外部に送らない点は、ツール系プロダクトとしての強みです。

### 7. テストと拡張の入口

`tests/` に `chord_rules` と `midi_tempo` の pytest があり、理論ルールの回帰テストが可能です。今後は `harmony` や `voice_leading` へのテスト追加で、リファクタ（特に `main_window.py` の分割）の安全性を高めやすい構成です。
